# P3 工单 · 命令中心接真实后端 + 六大域下钻页(把引擎变成可用产品)

> 状态:待开发(Maestro 拟单,创始人已 OK 推进)。负责人:工程师。审核:Maestro 逐行审。合并:创始人授权(可由 Maestro 代合并)。
> 依赖:P0(演示)、P1(会话鉴权)、P2(动作回写)均已合并进 main。基于 docs/01(命令中心/角色)、docs/09(建议)、docs/10(六大域)、packages/web 现有原型与 messages/{zh,en,ja}.json。

## 背景
后端闭环已全通(感知→验证→建议→批准→执行→撤销,六大域)。但前端命令中心目前还是原型/静态演示。P3 把它**接上真实后端 + 实时**,并补齐**六大域下钻页**,让店长能真正操作,也能对外演示/销售。

## 目标
店长登录后进入命令中心,看到**真实数据 + 实时更新**,能在界面上**批准/撤销**建议并看到动作结果,并能点进任一域看**下钻明细**。全程三语、默认中文、沙箱不崩。

## 范围
1. **先清一个后端小硬化(P2.1,快)**:动作执行侧幂等改为"**先抢幂等槽位(写 action_log ON CONFLICT)再执行副作用**",与撤销侧对称;补一个并发竞态测试。(来自 PR #12 评审的小跟进,非阻塞但顺手清掉。)
2. **登录接入**:管理端用 P1 的会话登录(本地/开发用 dev-gated mock 登录),会话驱动 tenant;命令中心所有请求带会话(Bearer/cookie),SSE 用 `?session=`。
3. **命令中心接真实后端 + 实时**:六大域态势瓦片(/overview)、排序建议流(/recommendations)、验证账本、实时沟通流、loop 条——全部从真实接口取数,SSE 实时刷新;断线自动重连。去掉硬编码/mock。
4. **批准/撤销交互**:每条建议的按钮调用 `POST /recommendations/:id/approve` 与 `/undo`(P2);界面显示执行结果(已执行 / 已拦截高风险 / 仅记录意图),已执行且可撤销的显示"撤销"入口;显示证据 chips + 置信度 + 理由。
5. **六大域下钻页**:点某个域瓦片 → 该域专属页,列出该域的对象/任务/告警/建议,带验证状态、证据、时间线(如患者流的 Room-3 冲突→验证故事可下钻查看)。
6. **三语与健壮性**:全程中/英/日、默认中文,动态内容也翻译;localStorage 走安全封装,禁用时不白屏。
