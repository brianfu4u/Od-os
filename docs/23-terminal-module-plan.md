# 「终端功能」模块 · 独立开发计划与实施时间表

> 模块定位:员工采集端——触发整个 agentic loop 的**源头**。采集、上传、下达、申请支援、打卡、扫描定位、截图、语音/录音,全部在这里发生,喂给后端交叉验证。
> 策略:**Web 优先**(手机浏览器/微信内置浏览器即可用,免 AppID/ICP,最快可测);**真微信小程序**作为产品化阶段的正式客户端(接同一套 API,后端零改动)。
> 负责人:外部工程师(客户端);创始人(STT 密钥、后续 AppID/ICP、试点诊所)。审核:Maestro 逐票逐行审。合并:创始人授权。

## 0 · 现状基线(已具备,不重复造)
- 浏览器员工终端 `/zh/console`(小程序替身)已上线:上报、上传(Choose File→手机相机)、扫码证据(替身)、运行建议扫描、重新核实。
- 后端接口就绪:`/reports`(clock_in/clock_out/event/task_update/scan,幂等)、`/uploads`(photo/screenshot/voice/doc,去 EXIF、签名下载)。
- 语音**上传**已支持;语音**转文字**未做(见 T4)。
