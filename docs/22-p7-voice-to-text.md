## P7 主体 · 语音转文字
### 目标
员工上传的语音(证据)自动**转写成文字**,文本显示在命令中心「实时沟通」流,并作为一条文字上报进入交叉验证管线;为将来的 LLM 抽取/语义分析预留干净输入。

### 范围
1. **触发**:当 `/uploads`(kind=voice)或带音频附件的上报入库后,**异步触发转写**(不阻塞上传返回)。
2. **STT 适配器(可插拔 seam)**:抽象一个 `Transcriber` 接口,dev/测试可接 OpenAI Whisper 或 Gemini 多模态;生产面向中国诊所建议接**腾讯云/阿里云 ASR**(在境内、合规、低延迟)。中文/日文/英文支持;语言可自动检测或按租户默认(默认中文)。凭证走环境变量,不进 git。
3. **落库与展示**:转写文本写到对应 Communication/Document 对象(保留原音频作证据,transcript 为派生字段),发事件 + 走 SSE,让命令中心**实时显示转写文本**;在沟通流里把它当作一条文字上报呈现。
4. **可靠性**:转写失败/低置信要**明确标记**(不臆造文本);可重试。原音频始终保留(即使转写失败,证据不丢)。
5. **面向未来**:transcript 字段结构化,便于(可选的)LLM 抽取层直接读取做声称抽取/语义分析——本票不实现 LLM 抽取,只把文本备好。

### 验收标准(DoD)
- [ ] 通过 `/zh/console`(或上传接口)传一段**中文语音** → 几秒内**转写文本出现在命令中心沟通流**,并可作为文字上报驱动后续。
- [ ] 中/日文均可转写;失败/低置信有明确标记,不臆造;原音频保留。
- [ ] STT 走可插拔适配器 + 环境变量凭证;异步、不阻塞上传。
- [ ] 只用合成数据、不接真实 PHI;多租户隔离不破。

### 依赖(需提供)
- 一个 STT 服务的 API Key(dev 可用 Whisper/Gemini;生产建议腾讯云/阿里云 ASR)。配到后端环境变量即可。
