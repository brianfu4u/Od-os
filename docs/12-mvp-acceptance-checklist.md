# 可演示 MVP · 验收清单(非技术版)

## 0 · 开始前:让工程师给你什么

这份清单在**命令中心接真数据 + 员工控制台演示台**做好后用。你不用装任何东西,只需让工程师给你:
- **两个网页入口**:①**命令中心**(店长看板,你主要盯它);②**员工控制台**(模拟微信小程序的小工具,用来发上报/传照片/扫码)。
- 或者:约个**共享屏幕**,工程师操作、你在命令中心这边看结果并对照本清单打勾。

> 真正的微信小程序此刻还没法在这套环境里跑(要小程序账号+备案),所以先用「员工控制台」代替员工手机——**验证的是同一条后端闭环**,将来真小程序接的是同一套接口。

**验收心法:** 你在员工控制台做一个动作 → **几秒内**命令中心出现对应变化,而且**说得清为什么**(带证据+置信度)。看到这个,就说明系统真的在「感知 → 验证 → 建议」。

## 1 · 先认识命令中心该有什么

打开**命令中心**,先确认这几块都在(对照之前的三语原型):
- [ ] 顶部**运营节拍**(总体健康分)+ 今日关键数字
- [ ] **六大域瓦片**:员工 / 患者 / 财务 / 营销 / 设备 / 库存
- [ ] **AI 副驾 · 指挥提示**(建议流):此刻可能空或有种子数据
- [ ] **交叉验证账本**:一条条「声称 → 证据 → 结论(已验证/冲突/待定)+ 置信度」
- [ ] **实时沟通 / 事件流**
- [ ] 右上角能切 **中文 / EN / 日本語**

✅ 都在 = 界面到位。下一步才是关键:让它**动起来**。

## 2 · 核心验收:冲突 → 已验证 闭环(最重要)

这是整套系统的灵魂,请一定亲手走一遍。

**第 1 步(员工控制台):** 发一条上报——任务选**「整理 3 号诊室」**,标记**完成**,但**不要传照片**(故意留破绽),提交。

**第 2 步(切到命令中心,几秒内看):**
- [ ] 建议流**自动冒出一条**:类似「3 号房『就绪』未验证 / 冲突」
- [ ] 写明**原因**(缺整理照片 / 用时过快)
- [ ] **置信度约 76%**,证据 chip 显示 💬 有、📷 缺
- [ ] **验证账本**新增一条 **冲突 CONFLICT** 记录

✅ 看到这些 = 「感知 → 验证」通了,而且**它能自己识破『假完成』**。

**第 3 步(回员工控制台):** 给这条任务**补传一张诊室照片**。

**第 4 步(再看命令中心,几秒内):**
- [ ] 同一条**自动重新验证** → 变成 **已验证 ~93%**
- [ ] 证据 chip 现在有 📷
- [ ] **账本出现第二条**(已验证)——**两条都在、旧的不消失**(这就是可审计)

✅ 看到它**自动翻绿**、账本留下完整两条 = **核心闭环 + 护城河(可审计交叉验证)真的通了。** 🎉

## 3 · 扫码 = 最强证据(接诊 / 用设备)

**员工控制台:** 做一次**扫码**——扫「患者就诊码」(接诊)或「设备资产码」(用设备)。

**命令中心该看到:**
- [ ] 出现一条**带时间戳**的扫码事件(谁、几点、扫了什么)
- [ ] 相关任务的验证里,这次扫码**作为强证据**被算进去(置信度更高)

✅ 说明「用手机扫码代替硬件传感器」成立——**带时间、证明在场、难造假**。

## 4 · 人在环:批准/忽略只记录,不乱动

**命令中心:** 对任意一条建议点 **批准 / 忽略 / 稍后**。
- [ ] 该建议状态变为 已批准 / 已忽略 / 已稍后
- [ ] **系统不会自动去改排班、发通知或动真实世界**(现在只记录你的决定)

✅ 说明**方向盘在你手里**——AI 只当副驾出主意,真正执行(动作回写)是后面 S4、且必须你批准。

## 5 · 多租户隔离 + 三语 + 收尾判定

**多租户(医疗关键,若演示里有两家诊所 A/B):**
- [ ] 以 A 诊所看时,**完全看不到 B 诊所的任何数据**;反之亦然

✅ 说明「一套系统卖给多家诊所、数据互不串」成立。

**三语:**
- [ ] 右上角切 中/EN/日,**整屏文字随之切换**,数字不变

## 6 · 遇到不对怎么办

- 哪一步**没出现应看到的东西**,或数字/文字不对:**截个图**、记下你点了什么,发给我或工程师。
- 我会判断是「配置/数值要调」(比如某任务的用时、必需证据)还是「真 bug」,并给出改法。
- 记住:**代码逻辑本身已在沙箱里全绿测过**(38 单测、多项集成、隔离 11/11);这一步主要是让**你亲眼确认体验**、以及把数值调到贴合你诊所实际。
